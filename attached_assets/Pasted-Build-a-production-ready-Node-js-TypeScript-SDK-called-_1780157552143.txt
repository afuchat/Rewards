Build a production-ready Node.js + TypeScript SDK called @afuchat/rewards.

VISION:
AfuRewards is an instant-install gamification SDK that works immediately after npm install with ZERO setup.

It provides:
- XP system
- Points system
- Levels
- Badges
- Streaks
- Leaderboards
- Event system

IMPORTANT RULES:
- NO payments
- NO wallets
- NO cash-out logic
- NO SMS or external verification
- ONLY gamification logic

---

CORE REQUIREMENT:
After installation:

npm install @afuchat/rewards

Developers must be able to do this instantly:

import AfuRewards from "@afuchat/rewards";

const rewards = new AfuRewards();

rewards.addXP("user1", 10);
rewards.unlockBadge("user1", "Starter");

NO configuration required.

---

ARCHITECTURE:

1. Default Mode (MUST BE DEFAULT)
- In-memory storage
- Auto-initialized
- No database setup
- No API keys
- Works instantly

2. Optional Cloud Mode (future-ready)
- PostgreSQL-ready structure
- Can be ignored for MVP

---

FEATURES:

XP SYSTEM:
- addXP(userId, amount)
- getXP(userId)

POINTS SYSTEM:
- addPoints(userId, amount)
- deductPoints(userId, amount)
- getPoints(userId)

LEVEL SYSTEM:
- getLevel(userId)
- calculateLevel(xp)

Level formula must be configurable but have a default.

BADGES SYSTEM:
- unlockBadge(userId, badgeId)
- getBadges(userId)

Badges include:
- name
- description
- icon (optional)

STREAK SYSTEM:
- updateStreak(userId)
- getStreak(userId)
- resetStreak(userId)

LEADERBOARD:
- getLeaderboard(type: "xp" | "points")
- return sorted top users

EVENT SYSTEM (IMPORTANT):
- on(eventName, callback)
- emit(eventName, payload)

Example:
rewards.on("user_posted", (user) => {
  rewards.addXP(user.id, 10);
});

Developers control all reward logic via events.

---

STORAGE:
- Default: in-memory object store
- Must auto-create user records when first used
- Must not require database setup in MVP

---

NON-GOALS:
Do NOT implement:
- payments
- wallets
- financial rewards
- SMS
- identity verification
- external APIs

---

OUTPUT REQUIREMENTS:
- Clean modular TypeScript code
- Exportable npm package
- Simple API design (Stripe-like simplicity)
- Include basic README with examples
- Include minimal working demo file

GOAL:
A developer should install and use the SDK in under 30 seconds with no setup.
